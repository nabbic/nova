Engagement Lifecycle

How a deal flows through the platform from first contact to final report

1
PE Firm creates a deal room
Names the target company, sets timeline, selects diligence scope (all 7 modules by default). Invites the seller via email with a secure link.
👤 Actor: Buyer
↓
2
Seller accepts & reviews scope
Seller gets a plain-English breakdown of what will be scanned. Can flag any modules they want to discuss with the buyer before proceeding. No surprises.
👤 Actor: Seller
↓
3
Seller connects systems
Guided wizard: authorise GitHub → AWS → Jira → etc. via OAuth. Each connector shows exactly what read-only access is requested. For on-prem components, one-click agent download with install instructions.
👤 Actor: Seller  ·  Estimated time: 15–30 min
↓
4
Agents run autonomously
All 7 agents execute in parallel where possible. Each writes structured findings to the store. Both sides see live progress. Estimated runtime: 2–6 hours depending on codebase size.
🤖 Actor: Platform  ·  No human input required
↓
5
AI synthesises findings
Synthesis layer reads all structured findings, generates executive summary with risk score, produces technical narrative per category, flags critical issues, and compiles the full report.
🤖 Actor: Platform
↓
6
Buyer reviews report & explores findings
Buyer gets notified. Live dashboard with exec summary + drilldown into every category. Can export full PDF report for investment committee. Can ask follow-up questions via AI chat grounded in the findings.
👤 Actor: Buyer (deal partner + technical advisor)
↓
7
Clarification requests (optional)
Buyer can raise specific questions from the report. These appear in the seller portal as structured requests. Seller responds in-platform — all Q&A is captured and appended to the report.
👤 Actor: Both sides
↓
8
Engagement closed
Report locked and archived. Buyer can share with advisors or investment committee. Seller's system access revoked. Deal room retained for reference.
Total elapsed: typically 1–3 days vs 4–6 weeks manually
